# `data.csv` column schema

One row per unique `person_id`. snake_case throughout. `config.json`
references these column names so `app.py` stays generic. Produced by
`_build/merge_data.py` from the `POST /v6/people` responses — the columns
mirror `_build/sumble_v6.build_person_row`.

**Identity:** `person_id` (int, PK), `name`, `current_title` (the endpoint's
`job_title`), `linkedin_url`, `sumble_url` (the endpoint's canonical person
link), `location`, `country` (display name).

**Org linkage:** `org_id`, `org_name`, `org_sumble_url` (the employer's
Sumble page link), `domain` (filled from the calibration-company org match;
blank for employers outside that set — it is the account-picker join key).

**Job function:** `job_function_name` (endpoint display name),
`job_function_slug` (the ICP slug from `spec.json` when the name matches an
ICP persona, else a slugified name; keys `config.job_function_ranges`).

**Seniority:** `job_level` (display name; "Individual Contributor" when none
detected), `job_level_rank` (0 = IC … 18 = CXO, from the baked map in
`_build/sumble_v6.py` — the endpoint returns names only),
`max_job_level_rank` (18, repeated on every row so the app normalises
without re-aggregating).

**Skills:** `matched_skills` (comma-separated ICP slugs; empty when none),
`skill_count` (0 when none). Source: the `organizations-duckdb` MCP query →
`_raw/skills.csv` (the v6 API has no per-person skills attribute).

**First-party flags (always present, default 0):**
- `is_crm_contact` — resolved from the uploaded CRM contact/lead list.
- `is_icp_gold` — resolved from the gold set (recommended: contacts
  role-attached to closed-won opportunities). Drives the Evaluation tab and
  the `_build/fit_weights.py` calibration.

**Account score (optional — present when the interview connected one):**
- `account_rank` (int, lower = better; blank when unmatched)
- `account_score` (float; blank when unmatched)

**1P signal columns (one raw + one norm per signal, present only when the
interview connected 1P signals; 0 for unmatched people):**
- `<signal>_raw` (float) — raw value
- `<signal>_norm` (float, 0–1) — p99-saturation-normalised; the scorer
  consumes this directly (see `template/README.md` → Scoring).

Columns referenced by `config.json` that aren't present in `data.csv`
must raise a startup error — never silently default to zero.

# `score.csv` (generated; never edit)

A superset of `data.csv`, sorted by rank: `rank` → every `data.csv` column →
`people_score` (0–100) → one `<Factor> (pts)` contribution column per
factor with non-zero total contribution (they sum exactly to
`people_score`). Regenerated by `score_sheet.py` on every Save, at startup,
and by the Download button.
