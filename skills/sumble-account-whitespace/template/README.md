# {customer_name} account whitespace

Generated by the
[sumble-account-whitespace](https://github.com/SumbleData/sumble-account-whitespace-skill)
skill. Self-contained, **zero-dependency** whitespace app that ranks
Sumble's universe of organizations by ICP fit and lets you tune signal
weights with sliders.

If the skill matched your CRM list to Sumble at run time, accounts
already in your CRM were excluded from the pool — every row here is
net-new prospect territory.

## Run

Any machine with Python 3.10+ — no `pip install`, no venv:

```bash
python app.py
```

Then open <http://localhost:8001>.

Override the port via `python app.py 9001` or `PORT=9001 python app.py`.

## What you see

A ranked **Whitespace** table with a per-row breakdown panel, search,
and pagination. The top bar has **Download whitespace config** (the
portable JSON spec — primary CTA) and a secondary **Download CSV** for
spot-checking the scored sample. If you uploaded a CRM, a **Subsidiaries**
tab holds orgs whose parent is a CRM account (land-and-expand targets);
the tab bar is hidden when there's only whitespace. No evaluation view.
To score accounts already in your CRM, run `/sumble-account-scoring`.

## How weights work

Two or three slider levels, each constrained to sum to 100%:

- **Section sliders** (when present) — split the budget between size /
  fit signals (ACV) and buying-window signals (Intent).
- **Category sliders** — distribute each section's budget across
  categories like ICP persona count, growth, density, relevant
  technologies, project × tech intent.
- **Within each category** (accordion) — distribute that category's
  budget across the individual signals inside it.

Final contribution per signal:
`(section% × category% × within%) / 10000 × normalised_value`.
Penalty multipliers (e.g. an IT-services penalty, or per-tag
multipliers) multiply the summed score by `(1 − pct/100)`.

All scoring runs client-side on slider input — no server round trip —
so 10k+ rows re-rank instantly.

## Saving weights

Click **Save** (or Cmd/Ctrl+S) to write `account-whitespace-weights.json`
AND `data.csv` (with the current `score` + `rank`, one per `org_id`) —
there is no auto-save, so slider changes show "Unsaved changes" until you
Save. The app re-opens with the saved weights next time you start it.

That file is also a self-describing scoring spec: it bundles the
scoring formula, the per-signal source columns, the data-source
metadata, and a pre-computed `effective_weight_pct` for each signal.
Hand it to a coding agent and it can re-implement the score in a
production pipeline without this app.

## Score a larger candidate list (public API)

The web app calibrates weights on the candidate pool. To score a larger
list of accounts, use `score_accounts.py` — a stdlib-only script that
re-implements your tuned score against Sumble's **public REST API**, so
it runs anywhere without this app or internal Sumble access:

```bash
export SUMBLE_API_KEY=...     # from sumble.com/account — or just run the
                             # command below in a terminal and it will prompt
                             # for the key (saved to ~/.config/sumble/api_key)
python score_accounts.py --accounts more_candidates.csv --out scored_accounts.csv
```

`more_candidates.csv` needs a `domain` and/or `name` column. The script
reads `account-whitespace-weights.json`, calls the public API per
account, and writes a ranked CSV.

Signals badged **SQL-only** in the app have no public-API equivalent
(e.g. YoY persona growth). The script drops them and re-normalises the
remaining weights, so its ranking is the *API-reproducible* model — very
close to, but not byte-identical to, the app's full model. Each account
costs a handful of API calls (see <https://docs.sumble.com/api> for the
credit table).

## Files

- `account-whitespace-weights.json` — the single source of truth: spec
  + tuned weights. The app reads it at startup and writes it on Save
  (no auto-save). Hand it to a coding agent to re-implement the
  score in a production pipeline.
- `data.csv` — the whitespace pool: top-N Sumble orgs by ICP fit, with
  any CRM matches already removed (if you uploaded a CRM list). One row
  per Sumble `org_id`.
- `score_accounts.py` — portable public-API scorer (above). Config-driven;
  same across customers, do not edit per customer.
- `app.py` — the HTTP server. Same across customers; do not edit
  per customer.
- `static/` — UI (logo, slider panel, table, breakdown).

## Regenerate

To pull fresh data or change which signals are included, re-run the
`sumble-account-whitespace` skill in Claude Code. The skill will
rewrite `account-whitespace-weights.json` and `data.csv` in place;
`app.py`, `score_accounts.py` and `static/` stay put.
