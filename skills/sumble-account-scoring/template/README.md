# {customer_name} account scoring

Generated by the
[sumble-account-scoring](https://github.com/SumbleData/sumble-account-scoring-skill)
skill. Self-contained, **zero-dependency** account-scoring app that ranks
companies by fit and lets you tune signal weights with sliders.

## Run

Any machine with Python 3.10+ — no `pip install`, no venv:

```bash
python app.py
```

Then open <http://localhost:8001>.

Override the port via `python app.py 9001` or `PORT=9001 python app.py`.

## Views

- **Your accounts** — every row in the calibration sample, ranked by
  fit. Prioritise outreach within this set.
- **Evaluation** — only shown when an ICP gold set is present
  (`is_icp_gold=1` rows). Buckets all scored accounts into deciles and
  shows hit rate, cumulative recall, and lift vs baseline. Live-recomputes
  on any slider move.

## How weights work

Two slider levels, both constrained to sum to 100%:

- **Category sliders** (top level) — distribute total fit "budget"
  across categories like ICP persona count, growth, density,
  complementary technologies, etc.
- **Within each category** (accordion) — distribute that category's
  budget across the individual signals inside it.

Final contribution per signal: `(category% × within%) / 100 × normalised_value`.
Multipliers (e.g. an IT-services penalty) multiply the summed score by
`1 − Σ pct × flag`.

All scoring runs client-side on slider input — no server round trip —
so 5k+ rows re-rank instantly.

## Saving weights

Weights **auto-save** as you adjust the sliders — there's no Save
button to click. Changes write to `account-scoring-weights.json`
~400ms after your last slider move, so the app re-opens with the
latest weights next time you start it.

That file is also a self-describing scoring spec: it bundles the
scoring formula, the per-signal source columns, the data-source
metadata, and a pre-computed `effective_weight_pct` for each signal.
Hand it to a coding agent and it can re-implement the score in a
production pipeline without this app.

To grab the spec without leaving the browser (e.g. to share or check
into git), click **Download scoring config** in the top bar. It exports
the current sliders as a JSON file matching the on-disk format. The
secondary **Download CSV** button below it dumps the scored sample for
spot-checking.

The web app calibrates weights on a **sample**. To score a larger account
list once weights are tuned, hand `account-scoring-weights.json` to the
separate Sumble scoring skill (it reproduces this score against Sumble's
public API). Signals badged **SQL-only** have no public-API equivalent and
are dropped by that scorer.

## Files

- `account-scoring-weights.json` — the single source of truth: spec +
  tuned weights. The app reads it at startup and auto-saves to it on
  every slider change. Hand it to a coding agent to re-implement the
  score in a production pipeline.
- `data.csv` — the calibration sample, one row per Sumble `org_id`,
  with Sumble + 1P signals already joined by `id_column`.
- `score_accounts.py` — portable public-API scorer (above).
  Config-driven; same across customers, do not edit per customer.
- `app.py` — the HTTP server. Same across customers; do not edit
  per customer.
- `static/` — UI (logo, slider panel, table, breakdown).

## Regenerate

To pull fresh data or change which signals are included, re-run the
`sumble-account-scoring` skill in Claude Code. The skill will rewrite
`account-scoring-weights.json` and `data.csv` in place; `app.py`,
`score_accounts.py` and `static/` stay put.
